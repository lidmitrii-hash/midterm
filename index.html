🎓 University Management System

REST API для управления университетской системой на Spring Boot

Java 17 Spring Boot 3.2.0 JPA / Hibernate H2 Database Maven
⚠️ Важная информация о GitHub Pages

Это Spring Boot приложение, которое требует Java сервер для запуска. GitHub Pages может хостить только статические HTML страницы.

Чтобы запустить полнофункциональное приложение, следуйте инструкциям по установке ниже.

📋 О проекте

Полнофункциональная система управления университетом с поддержкой:

👨‍🎓 Студенты

С персональными профилями (One-to-One)

📚 Курсы

С множественными уроками (One-to-Many)

🔗 Назначения

Студентов на курсы (Many-to-Many)

💾 Cascade

Автоматическое сохранение связанных данных

🗄️ Модель базы данных

Student (1) ←→ (1) StudentProfile
Student (M) ←→ (M) Course
Course (1) ←→ (M) Lesson

Entities

Entity Поля Связи
Student id, firstName, lastName, email (unique) @OneToOne с StudentProfile
@ManyToMany с Course
StudentProfile id, address, phone, birthDate @OneToOne с Student
Course id, title, credits @ManyToMany с Student
@OneToMany с Lesson
Lesson id, topic, duration @ManyToOne с Course

🚀 Установка и запуск

Предварительные требования

  • Java 17 или выше
  • Maven 3.6+ (или используйте встроенный в IDE)
  • IntelliJ IDEA (рекомендуется)

Быстрый старт

1. Клонируйте репозиторий
git clone https://github.com/lidimitrii/midterm.git cd midterm
2. Соберите проект
mvn clean install

Или в IntelliJ IDEA: View → Tool Windows → Maven → Reload All Maven Projects 🔄

3. Установите Lombok Plugin (IntelliJ IDEA)

File → Settings → Plugins → Marketplace → Найдите "Lombok" → Install → Restart

4. Включите Annotation Processing

File → Settings → Build, Execution, Deployment → Compiler → Annotation Processors

✓ Enable annotation processing → OK

5. Запустите приложение
mvn spring-boot:run

Или в IntelliJ IDEA: ПКМ на UniversityApplication.java → Run

6. Откройте браузер
http://localhost:8080

🔌 REST API Endpoints

Student API

POST /students

Создать студента с профилем

{ "firstName": "Иван", "lastName": "Иванов", "email": "ivan@example.com", "profile": { "address": "Москва, ул. Примерная, 1", "phone": "+7 999 123-45-67", "birthDate": "2000-01-15" } }
POST /students/{studentId}/courses/{courseId}

Назначить студента на курс

GET /students/{id}

Получить студента со всеми связями

Course API

POST /courses

Создать курс с уроками

{ "title": "Java Programming", "credits": 5, "lessons": [ { "topic": "Introduction to Java", "duration": 90 }, { "topic": "OOP Principles", "duration": 120 } ] }
POST /courses/{courseId}/lessons

Добавить уроки к существующему курсу

GET /courses/{id}

Получить курс со всеми уроками

✅ Реализованные требования

Task 1: Entity Mapping ✓

  • ✅ Все JPA аннотации (@OneToOne, @ManyToMany, @OneToMany)
  • ✅ Правильная настройка mappedBy, @JoinColumn, @JoinTable
  • ✅ Cascade types настроены корректно
  • ✅ Предотвращение JSON рекурсии (@JsonManagedReference, @JsonBackReference)

Task 2: Repository Layer ✓

  • ✅ StudentRepository extends JpaRepository
  • ✅ CourseRepository extends JpaRepository
  • ✅ LessonRepository extends JpaRepository

Task 3: Data Persistence Logic ✓

  • ✅ Сохранение Student → автоматически сохраняет StudentProfile
  • ✅ Назначение Student на Course с bidirectional связью
  • ✅ Добавление Lessons к Course с cascade
  • ✅ Удаление Student → Profile удаляется, Course остаётся

Task 4: REST API ✓

  • ✅ POST /students
  • ✅ POST /students/{studentId}/courses/{courseId}
  • ✅ GET /students/{id}
  • ✅ POST /courses
  • ✅ POST /courses/{courseId}/lessons
  • ✅ GET /courses/{id}
